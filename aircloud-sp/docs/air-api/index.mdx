Air API is an API-based workload that lets you quickly test and integrate supported models within a project. It's a great fit when you want fast integration without managing infrastructure directly.

## What you can do with Air API

- Create an API workload inside a project
- Choose a supported model
- Select an API access endpoint
- Issue a secret key
- Run a connectivity test with code samples
- Test in the Playground (Web UI)
- Consider expanding to Air Container using the same configuration

## Basic flow

1. Select Air API in the project overview
2. Choose a model
3. Create a secret key
4. Test with code samples
5. Test in Playground or integrate into your app

## Quickstart: Get started with Air API

Air API is a good fit when you want to quickly test and integrate model APIs in a project.

### 1. Select Air API in the project overview

After creating your organization and project, go to the project overview and select **Air API**.

### 2. Choose a model

Create an API workload in your project and select the AI model you need. Supported models are continuously added — if the model you need is not available, you can request it through feedback.

### 3. Issue an API key

Select an API access endpoint and create a secret key. The secret key cannot be viewed after creation, so make sure to copy and store it securely. If you lose it, you will need to generate a new one.

### 4. Test with a code sample

After receiving your key, return to the API screen and find the code sample. Replace `your-api-key` in the sample with your issued secret key to run a connectivity test.

### 5. Test in the Playground

Select the Playground to test the model directly in the Web UI. You can also choose to deploy with Air Container using the same configuration, allowing you to scale to a more stable service environment tuned to your required latency and GPU specs.

## When it's a good fit

- You want to integrate model APIs quickly
- You want to validate a PoC/prototype fast
- You want to start API-first before deploying containers
