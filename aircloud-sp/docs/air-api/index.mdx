Air API is an API-based workload that lets you quickly test and integrate supported models within a project. It’s a great fit when you want fast integration without managing infrastructure directly.

## What you can do with Air API

- Create an API workload inside a project
- Choose a supported model
- Select an API access endpoint
- Issue a secret key
- Run a connectivity test with code samples
- Test in the Playground (Web UI)
- Consider expanding to Air Container using the same configuration

## Basic flow

1. Select Air API in the project overview
2. Choose a model
3. Create a secret key
4. Test with code samples
5. Test in Playground or integrate into your app

## When it’s a good fit

- You want to integrate model APIs quickly
- You want to validate a PoC/prototype fast
- You want to start API-first before deploying containers
