You can deploy containers in two ways: from a **template**, or starting **from scratch** using a custom Docker image.

## Deploy from a template

Templates are pre-configured containers with popular frameworks and models. They let you get running without any Docker knowledge.

### 1. Select a project

From the AirCloud console, open a project or create a new one.

![Select a project](/images/air-container/select-project.png)

### 2. Click Deploy

Click **+ Deploy** to open the deployment options screen.

### 3. Select a template

Choose **Template** and select the framework or model you want to deploy.

![Select a template](/images/air-container/select-template.png)

## Deploy from scratch

Deploy a container from a custom Docker image you've prepared.

### 1. Select a project

From the AirCloud console, open a project or create a new one.

### 2. Click Deploy

Click **+ Deploy** to open the deployment options screen.

### 3. Select From Scratch

Choose **From Scratch** to configure your own container.

### 4. Configure resources

Set the GPU type, number of replicas, and other resource options.

#### GPU type

Select the GPU instance type that fits your workload. See the [Infrastructure and Resource Types](/docs/reference/infrastructure-and-resource-types) page for available instances and VRAM guidance.

#### Replicas

Set the number of replicas to run simultaneously.

AirCloud applies a **quota limit** on the number of replicas you can rent. If you need more than your current quota allows, contact an administrator to request an increase.

If your quota is sufficient but the GPU model you've selected is in high demand, you may not be able to secure the full number of replicas immediately. In that case, use **Join Waitlist** to register for the waitlist — you'll be notified when replicas become available.

#### Ports

Specify the ports your container needs to expose.

#### Environment variables

Add any environment variables your container requires.

### 5. Set the container image

Enter the Docker image URL. Private registries are supported — add your registry credentials in project settings.

### 6. Configure storage (optional)

Attach a persistent volume to retain data across restarts. See the [Persistent Volume](/docs/air-container/shared-storage) guide.

### 7. Review and deploy

Review your configuration and click **Deploy**. The container will transition to **Running** state once it starts successfully.
