This page walks through the standard flow to deploy a container using Air Container.

## 1. Select a project

Choose the project where you want to deploy your container. The project overview also shows endpoint status and current deployments.

## 2. Start creating a container

In the Air Container menu, click **+ Create** to open the container creation flow. Enter a container name and proceed.

## 3. Configure Container Image

Provide the following:

- Container image
- Registry provider
- Registry access type (public/private)
- If private: username + password/token

Common registry providers include:

- GCP Artifact Registry
- AWS ECR
- GitHub Container Registry
- Docker Hub

## 4. Configure Resources

Configure:

- Cloud Type
- Instance type
- Enable Autoscaling (optional)
- Replica count

We recommend balancing cost and latency based on your workload.

## 5. Advanced settings

Optionally configure:

- Start command
- Port
- Health check URL
- Max wait time
- Boot wait time
- Environment variables
- Endpoint ID

## 6. Review and deploy

Review the configuration and deploy.

Initial pull time depends on image size and registry performance. Pulled images may be cached on the node for faster subsequent starts.

## 7. Start the container

After deployment, the container may appear as **Pending**.

<Note>
  Pending state is not billed. To run workloads, click **Start container** to switch into a running state.
</Note>
