You can connect to a running container via SSH using a registered public key.

## Register SSH keys in the console

### 1. Go to SSH Key settings

In the console, open **Settings** and select **SSH Keys**.

### 2. Register your public key

Click **+ Add SSH Key**, paste your public key, and save.

Registered keys are available across all containers in the project.

## SSH Key Per-Container Application

You can apply a registered SSH key to specific containers only, or enable and disable keys without deleting them.

### Apply to a specific container only

In the container detail view, open **SSH Keys** and select the keys you want to enable for that container. Keys not selected here will not be accepted even if they are registered at the project level.

### Enable or disable without deleting

You can temporarily disable a key without removing it. This lets you revoke access quickly and restore it later without re-registering the key.

<Note>
  SSH key changes take effect after the container is restarted.
</Note>

## Connect via SSH

Once a key is registered and applied, use the SSH command shown in the container detail view to connect.

```bash
ssh -i ~/.ssh/your-private-key <user>@<host> -p <port>
```

The host and port are shown on the container detail page.
