A quickstart guide for new AirCloud users. In AirCloud, you first create an Organization and Project, then start your AI workload using either Air API or Air Container.

## Before you start

Before using AirCloud, complete sign-up and onboarding.

### 1. Select Get Started and choose a service

On the AirCloud homepage, click **Get Started**. In the service selection step, you can choose either **Run AI workloads** or **Provide resources**.

![Get Started and service selection](/docs/images/quickstart/quickstart-03.png)

- If you want to use the cloud, select **Go to User Dashboard** to move to the cloud console.
- If you want to register resources as an AirCloud provider, **Join the Waitlist** and we will contact you in order.
- If you want to try Air API, you can browse available models in the **AI Model** tab.

<Info>
  Models available via Air API will be continuously updated. See [Supported Models](https://docs.aieev.com/models/) for the latest information.
</Info>

### 2. Sign up and sign in

You can sign up using email or social login.

![Login Kor](/images/login-kor.png)

The currently supported sign-up methods are:

- Email
- GitHub

If you have issues signing up or logging in, request help via [**Contact Support**.](https://www.aieev.com/contact)

### 3. Complete onboarding

Complete the following three steps to get started with AirCloud.

1. **Create an Organization**
   Choose an organization name that best fits your use case, such as a company, research lab, or project. You can also invite collaborators to join your organization.
   ![온보딩 - 조직 생성](/docs/images/quickstart/quickstart-05.png)
   <Note>
     Each account can belong to only one organization. Once you create an organization, you cannot be invited to another one. If you need to change your organization, please [contact us](https://www.aieev.com/contact).
   </Note>
2. **Create a Project**
   Create a project for each service or development project you manage. A single project can contain multiple endpoints.
   ![프로젝트 생성 화면](/docs/images/quickstart/quickstart-08.png)
3. **Add Credits** _(Optional)_
   After completing the onboarding process, you will be redirected to the Console. Add credits to start using AirCloud services.
   ![크레딧 등록 화면](/docs/images/quickstart/quickstart-07.png)

AirCloud onboarding consists of the following three steps:

1. Create an Organization
   ![Onboarding - Create organization](/docs/images/quickstart/quickstart-05.png)
   An Organization name can represent a company, lab, or project. You can also add collaborators to an Organization.
2. Create a Project
   ![Create a project](/docs/images/quickstart/quickstart-08.png)
   A Project can represent a service you are building or a development effort. You can create multiple endpoints within a single Project.
3. Add credit (optional)
   ![Add credit](/docs/images/quickstart/quickstart-07.png)

After completing onboarding, go to the console to follow the first-time guide and start using the service.

## Choose your starting path

You can start with either of the following options:

## Invite team members (optional)

![Invite team members](/docs/images/quickstart/quickstart-invite-team.png)

AirCloud supports inviting members to your Organization and Project via email. Please keep the following in mind:

- Team members must sign up with the **same email address** that received the invitation.
- If you accept an invitation while logged into another account, you may see errors. On the same device, we recommend logging out first.

Owners can change all settings. Members can view organization settings, but cannot change billing-related settings.
