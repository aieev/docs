This page summarizes the Organization/Project structure, member management, and the credits/billing model you’ll encounter when getting started with Aircloud.

## Onboarding flow

Aircloud onboarding typically follows this sequence:

1. Create an Organization
2. Create a Project
3. Add credits (optional during onboarding)

After onboarding, you can create and manage workloads per project in the console.

## Organization and Project relationship

An **Organization** is the top-level administrative unit (team/company). A **Project** is the unit where you operate workloads. Air API and Air Container are both created and managed inside projects.

## Inviting members

Members can join an Organization via email invitation.

- Sign up using the **same email address** that received the invitation.
- If an invite link fails while you’re logged in with another account, log out and try again on the same device.

## Credits and payments

Depending on the resources and workloads used in a project, you may need credits or valid payment information.

Even if credits appear optional during onboarding, we recommend confirming billing readiness before running real workloads.

## Operational recommendations

- Separate projects for API testing vs. production workloads
- Define roles and access boundaries for each member
- Review usage and billing regularly
- Confirm credits/payment status before deployments
