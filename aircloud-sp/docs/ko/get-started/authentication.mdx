AirCloud의 모든 API는 시크릿 키 기반으로 인증합니다. 키를 발급하고 요청 헤더에 포함하면 Air API와 Air Container API 모두 동일한 방식으로 인증됩니다.

## API 키 발급

콘솔 좌측 메뉴에서 **API 키**를 클릭하고, **새 시크릿 키 생성**을 선택합니다.

<img src="/images/auth/api-key-list.png" alt="API 키 목록 — 새 시크릿 키 생성 버튼" />

키 생성 모달에서 아래 항목을 설정합니다.

<img src="/images/auth/create-secret-key.png" alt="새 시크릿 키 생성 모달 — 이름, Air-API 액세스, 컨테이너 액세스" />

| 항목 | 설명 |
|------|------|
| **이름** | 키의 목적과 사용 컨텍스트를 알아보기 쉬운 이름. 향후 관리가 쉬워집니다 (선택사항) |
| **Air-API 액세스** | 이 키로 호출할 수 있는 Air-API 엔드포인트를 지정합니다. 특정 모델만 허용하려면 목록에서 선택하세요 (선택사항) |
| **컨테이너 액세스** | 이 키로 접근할 수 있는 컨테이너를 지정합니다. 특정 컨테이너만 허용하려면 목록에서 선택하세요 (선택사항) |

<Warning>
  시크릿 키는 생성 직후 한 번만 확인할 수 있습니다. 반드시 복사해 안전한 곳에 보관하세요. 분실한 경우 해당 키를 삭제하고 새로 발급해야 합니다.
</Warning>

## 인증 헤더

모든 API 요청에는 `Authorization` 헤더에 시크릿 키를 포함해야 합니다.

```
Authorization: Bearer YOUR_API_KEY
```

## 키 보안 권고

시크릿 키를 코드에 직접 작성하지 마세요. 저장소에 커밋되면 외부에 노출될 수 있습니다. 환경 변수로 관리하는 것을 권장합니다.

<CodeGroup>
```bash .env
AIRCLOUD_API_KEY=YOUR_API_KEY
```

```python Python
import os
api_key = os.environ["AIRCLOUD_API_KEY"]
```

```javascript Node.js
const apiKey = process.env.AIRCLOUD_API_KEY;
```
</CodeGroup>

## 키 분실 또는 유출 시

콘솔 **API 키** 페이지에서 해당 키를 즉시 삭제하고 새로운 키를 발급받으세요. 키가 유출된 경우 삭제 전까지 해당 키로 API에 접근할 수 있으므로, 확인 즉시 삭제하는 것을 권장합니다.
