AirCloud를 처음 사용할 때 가장 먼저 이해하면 좋은 핵심 개념을 정리한 문서입니다.

<Info>
  처음이라면 [Quickstart](/docs/ko/quickstart)에서 전체 흐름을 먼저 확인한 뒤, 이 문서를 읽는 것을 권장합니다.
</Info>

## Organization(조직)

Organization은 사용자와 프로젝트를 묶는 상위 단위입니다. 하나의 팀, 회사, 또는 운영 주체를 기준으로 Organization을 생성하며, 멤버 초대와 권한 관리도 Organization 단위로 이루어집니다.

<Warning>
  하나의 계정은 1개의 Organization에만 속할 수 있습니다. Organization 생성 이후에는 다른 Organization의 초대를 받을 수 없으며, 소속 조직 변경이 필요한 경우 관리자에게 별도로 문의해 주세요.
</Warning>

## Project

Project는 실제 워크로드와 리소스를 관리하는 작업 단위입니다. Air API, Air Container, 사용량, 엔드포인트, 모니터링 정보는 모두 프로젝트 단위로 구분됩니다. 하나의 Organization에는 여러 개의 Project를 만들 수 있습니다.

## Air API

Air API는 모델을 API 형태로 빠르게 연동하고 테스트할 수 있는 방식입니다. 프로젝트 안에서 지원 모델을 선택하고 시크릿 키를 발급받은 뒤, 코드 샘플 또는 Playground를 통해 바로 테스트할 수 있습니다.

## Air Container

Air Container는 사용자가 직접 준비한 컨테이너 이미지를 배포하는 방식입니다. 런타임, 의존성, 포트, 환경 변수, 오토스케일링, Replica 수 같은 배포 옵션을 세밀하게 제어해야 하는 경우에 적합합니다.

## Endpoint

Endpoint는 배포된 API 또는 컨테이너 서비스에 접근하기 위한 실행 단위입니다. 사용자는 프로젝트 안에서 엔드포인트 상태, 사용량, 로그, 스케줄링 설정을 확인할 수 있습니다.

## Replica

Replica는 동일한 워크로드 인스턴스 수를 의미합니다. 트래픽이 많아질수록 Replica 수를 늘려 더 많은 요청을 처리할 수 있으며, Autoscaling을 통해 자동으로 조정할 수도 있습니다.

## Autoscaling

Autoscaling은 트래픽이나 설정 조건에 따라 Replica 수를 자동으로 조정하는 기능입니다. 최소 Replica 수와 최대 Replica 수를 함께 설정하여 비용과 성능을 균형 있게 운영할 수 있습니다.

## AirCloud Zero와 AirCloud Plus

AirCloud Zero와 AirCloud Plus는 AirCloud가 제공하는 두 가지 인프라 옵션으로, 노드 구성 방식과 자원 할당 방식에서 근본적인 차이가 있습니다.

**AirCloud Zero**는 PC방, 채굴 업체, 유휴 컴퓨팅 자원을 보유한 개인 등 다양한 참여자의 자원을 크라우드소싱 방식으로 모아 구성한 분산 클라우드 풀입니다. Spot 방식으로 자원이 할당되기 때문에 AirCloud Plus 대비 낮은 비용으로 이용할 수 있지만, 자원 회수 시 인터럽트가 발생할 수 있어 중단을 허용할 수 있는 워크로드에 적합합니다. 현재 출시 준비 중이며, 이른 시일 내 제공될 예정입니다.

**AirCloud Plus**는 AIEEV가 직접 운영·관리하는 전용 AI 연산 노드로 구성된 분산 클라우드 풀입니다. On-demand 및 Reserved 방식으로 자원이 할당되어 인터럽트 없이 안정적으로 점유할 수 있습니다. 중단 없는 프로덕션 서비스나 예측 가능한 성능이 중요한 워크로드에 적합합니다.

<Note>
  🛡️ **AirCloud Plus — ISO 27001 인증 획득**

  엔터프라이즈급 보안 및 컴플라이언스 요구사항이 있는 워크로드에 적합합니다.
</Note>

| 항목 | AirCloud Zero | AirCloud Plus |
| :-: | :-: | :-: |
| 인프라 구성 | 유휴 자원 크라우드소싱 (PC방, 채굴 업체 등) | AIEEV 직접 운영하는 전용 AI 연산 노드 |
| 할당 방식 | Spot — 인터럽트 발생 가능 | On-demand / Reserved — 안정적 점유 보장 |
| 상대 가격 | 더 저렴 | Zero 대비 높음 |
| 적합한 워크로드 | 비용 우선, 중단 허용 가능한 작업 | 중단 없는 프로덕션, 안정성 우선 |
| 보안 인증 | — | 🛡️ ISO 27001 |
| 출시 상태 | 출시 예정 | 현재 제공 중 |
