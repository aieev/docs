Aircloud는 AI inference workload를 빠르고 유연하게 배포하고 운영할 수 있도록 설계된 분산형 AI cloud platform입니다. 사용자는 API 기반으로 모델을 빠르게 연동할 수도 있고, 컨테이너 단위로 애플리케이션과 런타임을 직접 배포해 원하는 환경을 구성할 수도 있습니다.

간단한 프로토타이핑부터 production 서비스까지, Aircloud는 서비스 목적과 운영 방식에 맞는 다양한 시작 경로를 제공합니다. 처음 사용하는 경우에는 먼저 Air API와 Air Container 중 적합한 시작점을 선택한 뒤, 서비스 요구사항에 따라 Aircloud 또는 Aircloud+ 인프라 옵션을 함께 검토하는 흐름을 권장합니다.

## Aircloud에서 시작하는 방법

  </Card>
</Columns>

## 인프라 옵션

<Columns cols={2}>
  <Card title="Aircloud" icon="cloud">
    비용 효율과 유연성이 중요한 워크로드를 위한 분산형 인프라 옵션입니다. 합리적인 비용으로 AI inference workload를 운영하려는 팀에 적합합니다.
  </Card>

  <Card title="Aircloud+" icon="shield-check">
    더 높은 수준의 안정성, 성능, 가용성, 운영 통제가 필요한 워크로드를 위한 전용 인프라 옵션입니다. 엔터프라이즈 환경이나 중요한 production workload에 적합합니다.
  </Card>
</Columns>

## 어떤 경우에 무엇을 선택하면 되나요?

### 배포 방식 정하기

<Columns cols={2}>
  <Card title="빠르게 시작하고 싶은 경우" icon="rocket">
    모델 API를 빠르게 연동하고 운영 부담을 줄이고 싶다면 [Air API](/docs/api-reference)가 적합합니다.
  </Card>

  <Card title="직접 제어하고 싶은 경우" icon="settings">
    컨테이너 이미지, 런타임, 의존성, 서비스 구성을 직접 관리하고 싶다면 [Air Container](/docs/air-container)가 적합합니다.
  </Card>
</Columns>

### 인프라 옵션 정하기

<Columns cols={2}>
  <Card title="비용 효율이 중요한 경우" icon="wallet">
    유연하고 비용 효율적인 인프라가 중요하다면 Aircloud가 적합합니다.
  </Card>

  <Card title="안정성이 중요한 경우" icon="badge-check">
    더 높은 안정성, 예측 가능한 성능, 전용 운영 환경이 중요하다면 Aircloud+가 적합합니다.

    <Note>
      🛡️ ISO 27001 Certified — 엔터프라이즈급 보안/컴플라이언스 요구사항이 중요한 워크로드에 적합합니다.
    </Note>
  </Card>
</Columns>
