Air Cloud는 중요한 이벤트를 OS의 네이티브 알림으로 발송할 수 있습니다. 콘솔 탭을 켜두지 않아도 즉시 확인할 수 있어, 배포 실패 같은 긴급 상황을 놓치지 않을 수 있습니다. 본 가이드는 브라우저, 운영체제, Air Cloud 콘솔 세 가지 단계에서 데스크톱 알림을 활성화하는 방법을 안내합니다.

<Info>
  현재는 일부 우선순위가 높은 이벤트만 OS 알림으로 발송됩니다 — 예를 들어 **엔드포인트 배포 실패**, **서비스 공지사항** 등이 해당합니다. 그 외 인앱 알림(잔액 변동, 멤버 이벤트 등)은 콘솔 내 알림 팝오버에서만 확인할 수 있으며 데스크톱에는 표시되지 않습니다.
</Info>

## A. 동작 방식

데스크톱 알림은 브라우저의 [Web Notifications API](https://developer.mozilla.org/ko/docs/Web/API/Notifications_API)를 기반으로 동작합니다. 알림이 화면에 표시되려면 다음 세 가지 권한이 모두 허용되어야 합니다.

1. **브라우저 권한** — 브라우저별, 기기별로 따로 허용해야 합니다.
2. **운영체제 권한** — OS에서 해당 브라우저의 알림 송출을 허용해야 합니다.
3. **Air Cloud 콘솔 설정** — 프로필 설정에서 데스크톱 알림 토글을 켜야 합니다.

위 세 가지 중 하나라도 비활성화되어 있으면 데스크톱 알림은 발송되지 않습니다. 알림은 브라우저가 실행 중인 동안에만 전달되므로, 브라우저를 완전히 종료한 상태에서는 콘솔에 다시 접속하기 전까지 새 알림이 표시되지 않습니다.

## B. 콘솔에서 알림 활성화하기

프로필 설정으로 이동해 **데스크톱 알림** 토글을 켭니다. 처음 활성화할 때 브라우저가 Air Cloud의 알림 권한을 요청합니다.

팝업에서 **허용**을 클릭하세요. 실수로 **차단**을 눌렀다면 브라우저는 같은 권한을 다시 요청하지 않습니다. 이 경우 아래 [차단된 권한 다시 허용하기](#차단된-권한-다시-허용하기) 섹션을 참고하세요.

토글 상태는 브라우저별, 기기별로 저장됩니다. 다른 노트북이나 다른 브라우저를 사용하면 그곳에서도 다시 활성화해야 합니다.

## C. OS 레벨에서 알림 허용하기

브라우저 권한이 허용되어 있더라도, 운영체제 설정에서 막혀 있으면 알림이 표시되지 않습니다. 사용 중인 브라우저가 OS에서 알림을 보낼 수 있도록 설정되어 있는지 확인하세요.

### macOS

1. **시스템 설정 → 알림**으로 이동합니다.
2. 목록에서 사용 중인 브라우저(Chrome, Safari, Edge 등)를 선택합니다.
3. **알림 허용**을 켭니다.
4. **집중 모드** 또는 **방해금지 모드**가 꺼져 있는지 확인합니다.

<Frame>
  <img
    src="/docs/images/macos-webnoti-settings.png"
    style={{ borderRadius: "0.5rem" }}
  />
</Frame>

### Windows

1. **설정 → 시스템 → 알림**으로 이동합니다.
2. **앱 및 다른 보낸 사람의 알림** 항목에서 사용 중인 브라우저를 찾습니다.
3. 해당 브라우저의 알림을 **켬**으로 설정합니다.
4. **집중 지원** / **방해금지 모드**가 꺼져 있는지 확인합니다.

## D. 차단된 권한 다시 허용하기

이전에 브라우저 팝업에서 **차단**을 선택했다면, Air Cloud 콘솔에서 권한을 다시 요청할 수 없습니다. 브라우저의 사이트 설정에서 직접 변경해야 합니다.

- **Chrome / Edge**: 주소창의 자물쇠 아이콘 → **사이트 설정** → **알림**을 **허용**으로 변경
- **Firefox**: 자물쇠 아이콘 → **연결이 안전함** → **추가 정보** → **권한** → 알림의 **차단**을 해제
- **Safari**: **Safari → 설정 → 웹 사이트 → 알림** → Air Cloud 항목을 **허용**으로 변경

<Frame>
  <img
    src="/docs/images/chrome-webnoti-settings.png"

</Frame>

권한을 변경한 후에는 Air Cloud 콘솔을 새로고침하고, **데스크톱 알림** 토글을 한 번 껐다가 다시 켜서 권한을 새로 연결하세요.

## E. 문제 해결

- **토글이 자꾸 꺼집니다.** 브라우저 권한이 **차단** 상태일 가능성이 높습니다. 위 섹션의 사이트 설정에서 권한을 재허용하세요.
- **토글은 켜져 있는데 알림이 안 옵니다.** OS 레벨 설정과 집중 모드 / 방해금지 모드를 확인하세요. macOS의 화면 공유, 발표 모드 상태에서도 알림이 차단됩니다.
- **인앱 알림은 보이는데 데스크톱 알림은 안 옵니다.** 우선순위가 낮은 이벤트는 의도적으로 OS 알림을 발송하지 않습니다. 페이지 상단의 안내를 참고하세요.
- **Safari에서는 아무것도 표시되지 않습니다.** 일부 구버전 Safari는 PWA로 설치된 사이트에 한해 웹 알림을 발송합니다. macOS에서 데스크톱 알림이 필요하다면 Chrome, Edge, Firefox 사용을 권장합니다.

## F. 지원되는 이벤트

| 이벤트 | 타입 | 데스크톱 알림 |
| --- | --- | --- |
| 엔드포인트 배포 실패 | `ENDPOINT` | 발송 |
| 서비스 공지사항 | `ANNOUNCEMENT` | 발송 |
| 잔액 충전 / 차감 | `BALANCE` | 인앱 전용 |
| 멤버 초대 / 가입 | `MEMBER` | 인앱 전용 |
| 그 외 엔드포인트 이벤트 | `ENDPOINT` | 인앱 전용 |

데스크톱 알림 대상 이벤트는 점진적으로 확대됩니다. 모든 이벤트는 OS 알림 발송 여부와 관계없이 콘솔 내 알림 팝오버에서 항상 확인할 수 있습니다.
