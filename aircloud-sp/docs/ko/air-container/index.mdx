Air Container는 사용자가 직접 준비한 컨테이너 이미지를 AirCloud에 배포하고 운영할 수 있는 워크로드입니다. 런타임, 의존성, 포트, 환경 변수, 리소스, 오토스케일링 설정을 직접 제어해야 하는 경우에 적합합니다.

## Air Container로 할 수 있는 일

## 이런 경우에 적합합니다

- 자체 모델 서버나 애플리케이션 이미지를 배포하려는 경우
- GPU 스펙과 Replica 수를 직접 설정해야 하는 경우
- Autoscaling과 Scheduled Scaling을 함께 운영하려는 경우
- Air API보다 높은 수준의 운영 제어가 필요한 경우

## 컨테이너 생명주기

컨테이너는 5가지 상태를 가지며, 실제 컴퓨팅이 수행되는 **실행중·업데이트중** 상태에서만 시간당 과금이 발생합니다.

| 상태 | 설명 | 과금 |
|---|---|---|
| 🟢 실행중 | 컨테이너가 정상적으로 실행 중인 상태 | 발생 |
| 🟡 업데이트중 | 설정 변경 등으로 컨테이너가 재시작 중인 상태 | 발생 |
| ⚫ 대기중 | 컴퓨팅이 수행되지 않는 대기 상태. 컨테이너 중지 시 이 상태로 전환됨 | 없음 |
| 🔴 배포실패 | 이미지 pull 실패 등 배포 과정에서 오류가 발생한 상태 | 없음 |
| 🔴 비정상 | 컨테이너가 실행 중이나 정상 동작하지 않는 상태 | 없음 |

<Warning>
영구 볼륨을 사용하지 않으면, 컨테이너를 중지하거나 재시작할 때 컨테이너 내부 데이터가 유지되지 않습니다. 데이터를 보존하려면 [영구 볼륨](/docs/ko/air-container/shared-storage)을 사용하세요.
</Warning>
