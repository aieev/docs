## 시작·정지·재시작

배포된 컨테이너는 5가지 상태를 가지며, 상태에 따라 과금 여부와 데이터 보존 방식이 달라집니다.

### 컨테이너 상태

| 상태 | 설명 | 과금 |
| --- | --- | --- |
| 🟢 **실행중** | 컨테이너가 정상적으로 실행 중인 상태 | 시간당 발생 |
| 🟡 **업데이트중** | 설정 변경 등으로 컨테이너가 재시작 중인 상태 | 시간당 발생 |
| ⚫ **대기중** | 컴퓨팅이 수행되지 않는 대기 상태 | 없음 |
| 🔴 **배포실패** | 이미지 pull 실패 등 배포 과정에서 오류가 발생한 상태 | 없음 |
| 🔴 **비정상** | 컨테이너가 실행 중이나 정상 동작하지 않는 상태 | 없음 |

실제 컴퓨팅이 수행되는 **실행중**과 **업데이트중** 상태에서만 시간당 과금이 발생합니다. 대기중, 배포실패, 비정상 상태에서는 과금이 발생하지 않습니다.

### 컨테이너 중지

컨테이너를 중지하면 **대기중** 상태로 전환됩니다.

- 컴퓨팅이 중단되어 과금이 멈춥니다.
- 대기중 상태에서 다시 시작하면 컨테이너가 재실행됩니다.

### 데이터 보존

영구 볼륨 사용 여부에 따라 데이터 보존 방식이 달라집니다.

| 상황 | 영구 볼륨 미사용 | 영구 볼륨 사용 |
| --- | --- | --- |
| 컨테이너 중지 후 재시작 | 데이터 유지되지 않음 | 데이터 유지 |
| 컨테이너 재시작 | 데이터 유지되지 않음 | 데이터 유지 |

<Warning>
  영구 볼륨을 사용하지 않으면 컨테이너를 중지하거나 재시작할 때 컨테이너 내부에 저장된 데이터가 모두 사라집니다. 데이터를 보존해야 한다면 반드시 영구 볼륨을 사용하세요.
</Warning>

영구 볼륨 설정 방법은 [영구 볼륨](/docs/ko/air-container/shared-storage) 문서를 참고하세요.

## 컨테이너 태그

태그를 사용하면 여러 컨테이너를 사용자가 정한 이름으로 묶어 관리할 수 있습니다. 컨테이너 수가 많아질수록 용도나 모델 계열별로 그룹화해 빠르게 필터링하는 데 유용합니다.

### 태그 부여

컨테이너 목록 상단의 **\+ 태그 부여** 버튼을 눌러 원하는 이름으로 태그를 만들고, 태그를 적용할 컨테이너를 선택합니다. 하나의 태그를 여러 컨테이너에 동시에 적용할 수 있습니다.

### 태그로 필터링

태그를 사용하면 여러 컨테이너를 사용자가 정한 이름으로 묶어 관리할 수 있습니다. 컨테이너 수가 많아질수록 용도나 모델 계열별로 그룹화해 빠르게 필터링하는 데 유용합니다.

### 태그 부여

컨테이너 목록 상단의 **\+ 태그 부여** 버튼을 눌러 원하는 이름으로 태그를 만들고, 태그를 적용할 컨테이너를 선택합니다. 하나의 태그를 여러 컨테이너에 동시에 적용할 수 있습니다.

### 태그로 필터링

태그 목록에서 특정 태그를 클릭하면 해당 태그가 부여된 컨테이너만 목록에 표시됩니다. **선택 해제**를 누르면 전체 목록으로 돌아옵니다.
