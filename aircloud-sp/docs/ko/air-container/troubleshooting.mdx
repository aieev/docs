배포 이후 문제가 발생한 경우, 아래 순서로 확인하세요.

## Runtime Logs 확인

배포 직후 문제가 발생한 경우 Runtime Logs를 먼저 확인하는 것이 가장 빠릅니다. 로그 화면에서 실시간 로그를 확인하거나 다운로드할 수 있습니다.

## 자주 확인할 항목

- 이미지 경로가 올바른지
- 비공개 레지스트리 인증 정보가 정확한지
- 포트 설정이 올바른지
- 시작 명령이 올바른지
- 헬스 체크 URL이 정상 응답하는지
- 환경 변수가 누락되지 않았는지

## 컨테이너 상태별 확인

### 대기중 상태

배포 직후 컨테이너가 **대기중** 상태인 경우에는 아직 실행 상태가 아닙니다.

<Note>
  대기중 상태에서는 과금이 발생하지 않으며, 실제 실행을 위해서는 **컨테이너 시작**이 필요합니다.
</Note>

### 배포 실패 상태

컨테이너가 **배포 실패** 상태인 경우 Runtime Logs에서 원인을 확인하세요. 이미지 경로 오류, 레지스트리 인증 실패, 시작 명령 오류 등이 주요 원인입니다.

<Note>
  배포 실패 상태에서는 과금이 발생하지 않습니다.
</Note>

### 비정상 상태

컨테이너가 **비정상** 상태인 경우 헬스 체크 URL이 정상 응답하는지 확인하세요. 컨테이너 내부 오류나 포트 설정 문제가 주요 원인입니다.

<Note>
  비정상 상태에서는 과금이 발생하지 않습니다.
</Note>

## 설정 변경

컨테이너 이름 변경을 제외한 대부분의 수정 사항은 컨테이너를 Stop한 뒤 변경할 수 있습니다. 설정 변경 전에는 현재 엔드포인트 상태를 먼저 확인하는 것을 권장합니다.
