배포 이후에는 Usage, Runtime Logs, Settings를 통해 서비스 상태를 확인하고 운영할 수 있습니다.

## Usage

Usage 화면에서는 현재 사용량과 서비스 상태를 확인할 수 있습니다. **Healthy Replicas Count**를 통해 현재 실행 중인 Replica 수를 확인할 수 있습니다.

## Runtime Logs

로그 화면에서는 실시간 최신 로그를 확인할 수 있으며, 다운로드 기능도 제공합니다. 로그는 최신 순(내림차순)으로 정렬됩니다. 로그는 최신 순(내림차순)으로 정렬됩니다.

<Note>
  배포 직후 문제가 발생하는 경우에는 로그를 먼저 확인하는 것이 가장 빠릅니다.
</Note>

아래 기준으로 필터링할 수 있습니다.

- **시간 범위**: 특정 구간의 로그만 조회
- **컨테이너**: 컨테이너 단위로 필터링
- **인스턴스 ID**: 특정 replica의 로그만 조회

로그에는 다음 정보가 포함됩니다.

아래 기준으로 필터링할 수 있습니다.

- **시간 범위**: 특정 구간의 로그만 조회
- **컨테이너**: 컨테이너 단위로 필터링
- **인스턴스 ID**: 특정 replica의 로그만 조회

로그에는 다음 정보가 포함됩니다.

- 시작 명령어 출력시작 명령어 출력
- 헬스 체크 상태헬스 체크 상태
- 오류 메시오류 메시지
- 모델 서버의 stdout/stderr모델 서버의 stdout/stderr

컨테이너가 실패한 경우에도 로그는 일정 기간 동안 보존되어 디버깅에 활용할 수 있습니다.

## Settings

컨테이너 이름 변경을 제외한 대부분의 수정 사항은 컨테이너를 Stop한 뒤 변경할 수 있습니다. 설정 변경 전에는 현재 엔드포인트 상태를 먼저 확인하는 것을 권장합니다.

## API 테스트

### 단일 요청 테스트

Postman 등 API 테스트 도구를 사용해 배포된 컨테이너의 엔드포인트가 정상 응답하는지 확인할 수 있습니다.

### 오토스케일링 부하 테스트

Locust를 사용해 트래픽 부하를 시뮬레이션하고 Replica 확장 동작을 검증할 수 있습니다. Usage 화면에서 Replica 수, 요청 처리율, 응답 시간 변화를 함께 모니터링하세요.

## 자주 확인할 항목

배포 직후 문제가 발생하는 경우 아래 항목을 순서대로 확인하세요.

- 이미지 경로가 올바른지
- 비공개 레지스트리 인증 정보가 정확한지
- 포트 설정이 올바른지
- 시작 명령이 올바른지
- 헬스 체크 URL이 정상 응답하는지
- 환경 변수가 누락되지 않았는지
