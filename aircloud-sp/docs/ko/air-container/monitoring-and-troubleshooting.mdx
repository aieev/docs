배포 이후 문제가 발생하면 아래 순서로 확인하세요. 런타임 로그와 실행현황은 [모니터링](/docs/ko/air-container/monitoring) 페이지에서 확인할 수 있습니다.

## Settings

컨테이너 이름 변경을 제외한 대부분의 수정 사항은 컨테이너를 Stop한 뒤 변경할 수 있습니다. 설정 변경 전에는 현재 엔드포인트 상태를 먼저 확인하는 것을 권장합니다.

## API 테스트

### 단일 요청 테스트

Postman 등 API 테스트 도구를 사용해 배포된 컨테이너의 엔드포인트가 정상 응답하는지 확인할 수 있습니다.

### 오토스케일링 부하 테스트

Locust를 사용해 트래픽 부하를 시뮬레이션하고 Replica 확장 동작을 검증할 수 있습니다. [실행현황](/docs/ko/air-container/monitoring)에서 Replica 수, 요청 처리율, 응답 시간 변화를 함께 모니터링하세요.

## 자주 확인할 항목

배포 직후 문제가 발생하는 경우 아래 항목을 순서대로 확인하세요.

- 이미지 경로가 올바른지
- 비공개 레지스트리 인증 정보가 정확한지
- 포트 설정이 올바른지
- 시작 명령이 올바른지
- 헬스 체크 URL이 정상 응답하는지
- 환경 변수가 누락되지 않았는지
