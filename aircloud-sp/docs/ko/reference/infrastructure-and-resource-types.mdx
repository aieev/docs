AirCloud에서 사용할 수 있는 인프라 유형과 GPU 인스턴스 스펙을 정리한 문서입니다.

## Cloud Type

컨테이너 배포 시 워크로드 특성에 따라 두 가지 Cloud Type 중 하나를 선택할 수 있습니다.

<CardGroup cols={2}>
  <Card title="AirCloud Zero" icon="zap">
    **Spot 인프라** — 준비 중

    비용 효율이 중요한 워크로드에 적합합니다. 가용 자원 상황에 따라 인스턴스가 회수될 수 있어, 중단에 강한 워크로드에 권장합니다.
  </Card>

  <Card title="AirCloud Plus" icon="server">
    **On-demand 인프라**

    AIEEV가 직접 운영하는 분산 클라우드입니다. 작업 중단 없이 안정적으로 워크로드를 운영할 수 있습니다.
  </Card>
</CardGroup>

## VRAM 기준 GPU 선택 가이드

모델을 올릴 때 VRAM이 부족하면 로드 자체가 불가능합니다. 아래는 모델 파라미터 크기별 최소 VRAM 기준입니다.

| 모델 크기 | 최소 VRAM (fp16 기준) | 권장 GPU |
| --- | --- | --- |
| ~7B | 14GB 이상 | RTX 4070 Ti Super (16GB) |
| ~13B | 26GB 이상 | RTX 4090 (24GB, 양자화 적용 시) |
| ~30B | 60GB 이상 | RTX 5090 (32GB, 양자화 적용 시) |
| ~70B | 140GB 이상 | RTX PRO 6000 (96GB, 양자화 적용 시) |

<Note>
  위 수치는 fp16 기준 대략적인 참고값입니다. 실제 VRAM 사용량은 배치 크기, 시퀀스 길이, 양자화 방식에 따라 달라집니다.
</Note>

## 선택 시 고려할 사항

- **모델 크기**: 로드하려는 모델의 파라미터 수와 정밀도에 따라 필요한 VRAM이 결정됩니다.
- **응답 속도**: VRAM이 클수록 추론 처리 속도가 빠르고, 더 큰 배치를 처리할 수 있습니다.
- **레플리카 수**: 트래픽이 많다면 Autoscaling 범위를 고려해 GPU 비용을 계획하세요.
- **배치 크기**: 동시 요청 수가 많을수록 더 많은 VRAM이 필요합니다.
