AirCloud 문서의 구조 변경, 신규 가이드 추가, 콘텐츠 개선 사항을 기록하는 페이지입니다.

## 2026-06

### Air Container

- **컨테이너 태그**: 사용자 정의 이름으로 여러 컨테이너를 묶어 그룹화하고 필터링하는 기능 안내 추가
- **'시작·정지·재시작' → '컨테이너 관리'로 통합**: 태그 기능 포함, 생명주기·과금 규칙 내용 재편
- **영구 볼륨 개선**: 오토스케일링 배포 지원, 생성 시 새/기존 볼륨 선택, 여러 배포 간 공유, 중지 상태에서 볼륨 활성화·교체 기능 안내 추가
- **레플리카 쿼터 제한**: 쿼터 초과 시 관리자 문의, GPU 수요 부족 시 대기 신청(waitlist) 안내 추가
- **SSH 키 컨테이너별 적용**: 특정 컨테이너에만 적용하거나 삭제 없이 활성화/비활성화하는 기능 안내 추가
- **웹 터미널 레플리카 선택**: 다중 레플리카 환경에서 접속 대상 선택, 레플리카 교체 시 자동 재연결 안내 추가

### Air API

- **모니터링 페이지 신규 추가**: 프로젝트 모니터링(Air API 탭) 지표 안내, 조직 사용량 관리에서 Air API 집계 확인 방법 추가
- **사용량 개선 반영**: 지출 집계 정확도 개선, 필터 일관성 정리, 차트·기록에 모델 표시 이름 추가 내용 반영
- **코드 예시 Shell 탭 추가**: Chat·TTS·Embedding 세 섹션 모두 Python / Node.js / cURL / Shell 4종으로 통일

### 레퍼런스

- **인프라 및 리소스 타입 페이지 신규 추가**: AirCloud Plus·Zero 비교 카드, GPU 스펙 테이블(VRAM·RAM·vCPU), 모델 크기별 VRAM 권장 가이드, 가격 안내 페이지 링크

### 콘솔

- **과금 — 환불 정책 섹션 추가**: 7일 이내 미사용 크레딧 환불 기준, 부가세 제외 공급가액 기준, 청구 이의 제기 안내, 이용약관 링크

### 지원

- **FAQ AccordionGroup 개편**: 기존 헤딩 나열 방식을 접기/펼치기 형태로 변경해 가독성 개선

### 시작하기

- **AirCloud란? 페이지 개편**: 배포 방식 선택(Air API·Air Container)과 인프라 옵션 선택(Zero·Plus) 카드 구조로 재편, 중복 섹션 제거
