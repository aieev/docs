Aircloud를 처음 사용하는 사용자를 위한 빠른 시작 가이드입니다. Aircloud에서는 먼저 조직과 프로젝트를 생성한 뒤, Air API 또는 Air Container 중 원하는 방식으로 AI workload를 시작할 수 있습니다.

## 시작 전 준비

Aircloud를 사용하기 전에 먼저 로그인과 온보딩을 완료해야 합니다.

### 1. Get Started 및 서비스 선택

Aircloud 홈페이지에서 Get Started를 눌러 시작합니다. 서비스 선택 단계에서는 AI 워크로드 실행 또는 자원 공급 중 하나를 선택할 수 있습니다.

자원 공급 서비스는 현재 클로즈드 베타로 운영되고 있으므로, 일반적인 Quickstart 흐름에서는 AI 워크로드 실행 기준으로 진행하는 것을 권장합니다.

### 2. 회원가입 및 로그인

이메일 가입 또는 간편 로그인을 통해 회원가입할 수 있습니다. 현재 지원되는 가입 방식은 다음과 같습니다.

- 이메일 가입
- GitHub 계정 로그인
- Google 계정 로그인

### 3. 온보딩 완료

Aircloud의 온보딩은 아래 3단계로 구성됩니다.

1. 조직 생성
2. 프로젝트 생성
3. 크레딧 등록 (선택사항)

온보딩을 완료한 뒤에는 콘솔에 접속하여 첫 사용자를 위한 안내 가이드를 확인하고 서비스를 시작할 수 있습니다.

## Air Container로 시작하기

Air Container는 직접 컨테이너 이미지를 배포하려는 경우에 적합합니다.

### 1. 프로젝트 선택

생성한 조직 안에서 컨테이너를 배포할 프로젝트를 선택합니다. 프로젝트 개요 페이지에서는 현재 엔드포인트 현황도 함께 확인할 수 있습니다.

모든 조직은 반드시 하나 이상의 프로젝트를 포함합니다.

### 2. 컨테이너 생성 시작

Air Container 메뉴에서 **+ 생성**을 눌러 컨테이너 생성 페이지로 이동합니다. 먼저 컨테이너 이름을 입력한 뒤 다음 단계로 이동합니다.

### 3. Container Image 설정

컨테이너 이미지 정보를 입력합니다. 입력 시 확인해야 할 항목은 다음과 같습니다.

- 컨테이너 이미지
- 레지스트리 공급자 선택
  - GCP Artifact Registry (레지스트리 지역 입력 필수)
  - AWS ECR (레지스트리 지역 입력 필수)
  - GitHub Container Registry
  - Docker Hub
- 레지스트리 액세스 유형 선택 (공개 또는 비공개)
- 비공개 레지스트리 선택 시 사용자명과 비밀번호 또는 토큰 입력

### 4. Resource 설정

사용할 인프라 유형과 리소스를 선택합니다. 설정 가능한 주요 항목은 다음과 같습니다.

- Cloud Type 선택
  - Aircloud+ (자사 보유 디바이스 기반 인프라)
  - Aircloud (개인, PC방, 공급 파트너사 기반 인프라)
- 인스턴스 타입 선택
  - RTX 4070 Ti Super
  - RTX 4070 Super
  - RTX 4090
  - RTX 5090
  - RTX PRO 6000
- Autoscaling 활성화 여부
- Replica 개수 설정

Autoscaling을 사용하는 경우 최소 유지 Replica 수와 최대 확장 가능 Replica 수를 함께 설정할 수 있습니다.

### 5. Advanced 설정

추가 옵션은 기본값이 있거나 필수 입력이 아닙니다. 필요한 경우 아래 항목을 설정할 수 있습니다.

- 시작 명령
- 포트
- 헬스 체크 URL
- 최대 대기 시간
- 부팅 대기 시간
- 환경 변수
- 엔드포인트 ID

### 6. Review 및 Deploy

입력한 배포 정보를 검토한 뒤 컨테이너를 배포합니다. 이미지 크기와 Registry 성능에 따라 초기 다운로드 시간은 달라질 수 있습니다.

한 번 다운로드된 이미지는 노드에 캐시되므로 이후에는 더 빠르게 실행될 수 있습니다.

### 7. 컨테이너 시작

배포 직후 컨테이너는 대기중 상태로 표시됩니다. 대기중 상태에서는 과금이 발생하지 않으며, 실제로 사용하려면 반드시 **컨테이너 시작**을 눌러 실행 상태로 전환해야 합니다.

## Air API로 시작하기

Air API는 프로젝트 안에서 모델 API를 빠르게 테스트하고 연동하려는 경우에 적합합니다.

### 1. 프로젝트 개요에서 Air API 선택

조직과 프로젝트를 생성한 뒤, 프로젝트 개요 화면으로 이동하여 **Air API**를 선택합니다.

### 2. 사용할 모델 선택

프로젝트 내에서 API 워크로드를 생성하고, 필요한 AI 모델을 선택합니다. 지원 모델은 지속적으로 추가될 예정이며, 필요한 모델이 없는 경우 피드백을 통해 요청할 수 있습니다.

### 3. API 키 발급

API 액세스 엔드포인트를 선택한 뒤 시크릿 키를 생성합니다. 시크릿 키는 생성 이후 다시 확인할 수 없으므로, 반드시 복사해 안전하게 보관해야 합니다. 분실한 경우 재발급이 필요합니다.

### 4. 코드 샘플로 연결 테스트

키를 발급받은 뒤 다시 API 화면으로 돌아가 코드 샘플을 확인합니다. 코드 샘플의 your-api-key 위치에 발급받은 시크릿 키를 입력해 연결 테스트를 진행합니다.

### 5. Playground에서 테스트

Playground를 선택하면 Web UI에서 직접 모델 테스트를 진행할 수 있습니다. 또한 동일한 configuration으로 Air Container 배포를 선택할 수 있으므로, 요구 latency와 GPU 스펙에 맞춰 더 안정적인 서비스 환경으로 확장할 수 있습니다.

## 모니터링 및 운영

배포 후에는 사용량과 상태를 모니터링할 수 있습니다.

### Usage

Usage 화면에서는 실행 중인 서비스의 사용량을 확인할 수 있습니다. Healthy Replicas Count를 통해 현재 활성화된 Replica 수를 확인할 수 있습니다.

### Runtime Logs

로그 화면에서는 다양한 보기 옵션으로 실시간 최신 로그를 확인할 수 있으며, 다운로드 기능도 제공합니다.

### Settings

컨테이너 이름 변경을 제외한 대부분의 수정 사항은 컨테이너를 Stop한 이후에 변경할 수 있습니다.

### Scheduled Scaling

시간 기반 스케줄링을 사용하면 예측 가능한 트래픽 패턴에 맞춰 지정된 시간에 Replica 수를 자동으로 조정할 수 있습니다. 예를 들어, 매일 오전 11시 30분부터 오후 6시 30분까지 트래픽이 증가하는 경우 다음과 같이 설정할 수 있습니다.

- 오전 11시 30분: Autoscale On, Min 1 / Max 3
- 오후 6시 30분: Autoscale Off, Replica 1

## 팀원 초대 (선택사항)

Aircloud에서는 이메일 초대 기능을 통해 멤버를 조직과 프로젝트에 초대할 수 있습니다. 초대 진행 시에는 아래 사항을 유의해야 합니다.

- 초대 메일을 받은 주소와 동일한 이메일로 회원가입해야 합니다.
- 기존 계정으로 로그인된 상태에서 초대 수락을 누르면 오류가 발생할 수 있으므로, 동일한 디바이스에서 작업할 경우 먼저 로그아웃한 뒤 진행하는 것이 좋습니다.
