## External API 확장 및 신규 모델 추가

External API가 전체 엔드포인트 라이프사이클 관리를 지원합니다. 추론을 위한 3종의 AI 모델이 플랫폼에 추가되었습니다.

**External API: 신규 엔드포인트**

5개의 신규 엔드포인트로 배포를 완전히 프로그래밍 방식으로 제어할 수 있습니다:

- **엔드포인트 목록 조회**: 접근 가능한 모든 엔드포인트를 상태 및 구성 정보와 함께 페이지네이션으로 조회합니다.
- **레플리카 목록 조회**: 활성 엔드포인트의 실시간 레플리카 상태를 확인합니다.
- **로그 파일 목록/내용 조회**: 디버깅을 위해 엔드포인트 로그 파일에 접근하고 내용을 다운로드합니다.
- **엔드포인트 설정 수정**: 비활성 엔드포인트의 런타임 설정(레플리카 수, 스케일링 구성)을 변경합니다.
- **인증 컨텍스트 조회**: API 키의 인증 범위와 권한을 확인합니다.

## Jupyter Notebook, Web IDE, 영구 볼륨 지원

GPU 워크로드를 위한 개발 환경과 스토리지 기능이 추가되었습니다.

**Jupyter Notebook 환경 지원**

GPU 기반 Jupyter Notebook 환경을 템플릿 형태로 제공하여, 별도의 설정 없이 바로 실행 가능한 개발 환경을 지원합니다.

- TensorFlow, PyTorch 등 주요 라이브러리 사전 구성
- GPU 리소스를 활용한 실험 및 모델 개발 가능
- 브라우저 기반 접근 지원
- Jupyter AI 기본 연동

**Web IDE (Code) 환경 지원**

브라우저에서 바로 사용할 수 있는 VS Code 기반 Web IDE 환경이 추가되었습니다.

- 로컬 개발 환경 없이 브라우저에서 코드 작성 및 실행 가능
- 컨테이너 기반 격리된 개발 환경 제공
- 개발 및 디버깅을 위한 기본 도구 지원
- 코드 어시스턴트 기본 연동

**Persistent Volume (영구 볼륨) 지원**

컨테이너 종료 이후에도 데이터를 유지할 수 있도록 영구 볼륨 기능이 추가되었습니다.

- 모델 체크포인트, 로그, 데이터 파일 지속 저장
- 컨테이너 lifecycle과 독립적인 데이터 관리
- 장기 작업 및 반복 실험에 적합한 환경 제공

</Update>

## External API 최초 출시

프로그래밍 방식의 엔드포인트 관리를 위한 AirCloud External API가 출시되었습니다. 터미널이나 CI/CD 파이프라인에서 직접 추론 엔드포인트를 제어하세요:

```bash
# 엔드포인트 상태 조회
curl -X GET https://external.aieev.cloud:5007/external/api/v1/endpoints/{id} \
  -H "Authorization: Bearer YOUR_API_KEY"

# 레플리카 스케일링
curl -X POST https://external.aieev.cloud:5007/external/api/v1/endpoints/{id}/scale \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"replica_count": 3}'
```

주요 기능:

- **엔드포인트 상태 조회**: 엔드포인트의 현재 상태, 구성, 헬스 정보를 확인합니다.
- **엔드포인트 시작/중지**: API를 통해 엔드포인트를 온디맨드로 시작하거나 중지합니다.
- **레플리카 스케일링**: 트래픽 요구사항에 맞게 활성 엔드포인트의 레플리카 수를 조정합니다.

## 대규모 TTS 용량 검증

프로덕션급 텍스트 음성 변환 워크로드에 대한 플랫폼 용량을 검증했습니다:

- 저지연 스트리밍 요구사항을 충족하는 **일일 80만 건 이상의 TTS 요청**을 검증했습니다.
- 소규모 파일럿에서 전체 프로덕션 트래픽으로의 단계적 마이그레이션 경로를 확인했습니다.
- 콜드 스타트 영향을 최소화하기 위한 캐싱 및 사전 프로비저닝 전략으로 대형 컨테이너 배포를 테스트했습니다.

## 관측성, 대시보드, API 키, UX 대규모 개선

포괄적인 관측성, 프로젝트 대시보드, 보안 강화를 제공하는 대규모 릴리즈입니다.

**관측성 및 로그**

런타임 로그를 검색, 필터링, 내보내기할 수 있습니다:

- **로그 다운로드/일괄 내보내기**: 단일 또는 일괄 내보내기로 로그를 다운로드하여 오프라인 분석이 가능합니다.
- **고급 로그 필터링**: 정규식 기반 검색 및 시간대별 필터링으로 더 빠른 트러블슈팅을 지원합니다.
- **원시 로그(JSON) 보기**: 포맷된 화면과 함께 가공되지 않은 로그 데이터를 확인할 수 있습니다.
- **스케일링 히스토리**: 오토스케일 이벤트, 타임아웃 트리거 변경, 에러로 인한 레플리카 조정을 시간순으로 추적합니다.

**대시보드**

사용량 가시성을 위한 프로젝트 및 조직 수준의 대시보드가 추가되었습니다:

- **프로젝트 대시보드**: 전용 뷰로 프로젝트 수준의 상태와 사용량을 모니터링합니다.
- **조직 사용량 분석**: 프로젝트, 엔드포인트, 인스턴스 타입별 사용량 및 비용 분포를 확인합니다.
- **확장된 요청 메트릭**: 누적 요청 수, 작업 에러율 등 운영 메트릭을 제공합니다.
- **거래 내역 필터링**: 타입 및 상태별로 결제 내역을 필터링합니다.

**UX 개선**

- **한국어 지원**: 한국어 UI가 추가되었습니다.
- **첫 사용자 온보딩**: 첫 엔드포인트를 배포하기 위한 신규 사용자 가이드 튜토리얼이 제공됩니다.
- **인앱 피드백**: 플랫폼 내에서 직접 피드백을 전달할 수 있습니다.

**보안 및 접근**

- **OpenAI 호환 API 키**: OpenAI 호환 형식의 API 키를 발급합니다. 키별로 접근 가능한 엔드포인트를 제어할 수 있습니다.
- **HTTPS 엔드포인트**: 모든 추론 엔드포인트에 HTTPS가 적용되었습니다.
- **경로 기반 엔드포인트 주소 체계**: 포트 기반에서 경로 기반 라우팅으로 전환하고, 커스텀 엔드포인트 URL을 지원합니다.

</Update>

## AirCloud 정식 출시

AirCloud가 정식 출시되었습니다. API 우선 워크플로우, 커스텀 엔드포인트, 스케줄 기반 스케일링을 제공하는 마일스톤 릴리즈입니다.

- **시간 기반 스케일링**: 시간대별로 최소 레플리카 수를 설정하거나 오토스케일링을 토글합니다. 트래픽 패턴에 맞추고 유휴 비용을 자동으로 절감합니다.
- **커스텀 엔드포인트 URL**: 시스템 생성 ID 대신 사용자 지정 URL 식별자로 더 깔끔한 연동과 공유가 가능합니다.
- **API Playground**: 코드 작성 없이 내장된 Playground에서 API를 인터랙티브하게 테스트하고 연동할 수 있습니다.
- **원클릭 클러스터 배포**: 템플릿 기반 자동 클러스터 배포로 반복적인 설정 작업을 줄입니다.
- **실행 가능한 에러 메시지**: 장애 화면에서 상세한 에러 안내를 제공하여 평균 해결 시간을 단축합니다.
- **안전한 추론 트래픽 (HTTPS)**: 모든 추론 요청에 대한 엔드투엔드 HTTPS.
- **리소스 관리 개선**: 프로덕션 워크로드를 위한 리소스 할당 및 운영 제어를 강화했습니다.

</Update>

## 사용량 인사이트, 오토스케일링 제어, 성능 튜닝

비용 가시성 강화, 더 스마트한 오토스케일링, 인프라 수준의 성능 강화를 제공합니다.

- **사용량 및 과금 대시보드**: 월별/일별 사용량 시각화로 크레딧 소비와 지출 추이를 한눈에 추적합니다.
- **오토스케일링 민감도 설정**: 워크로드 특성에 맞게 오토스케일 반응성(Heavy / Normal / Light)을 설정합니다. 버스트 추론부터 안정적 처리량까지 대응합니다.
- **배포 템플릿 재사용**: 기존 배포 구성을 복사하여 동일한 설정으로 새 엔드포인트를 빠르게 생성합니다.
- **예측 가능한 스케일링 동작**: 다양한 부하 패턴에서 더 일관된 오토스케일링을 위해 스케일 인/아웃 정책과 감쇠를 튜닝했습니다.
- **고동시성 전송**: 피크 트래픽에서의 대용량 파일 전송 및 동시 요청 처리 성능을 튜닝했습니다.

</Update>

## 무중단 운영 강화

플랫폼 업데이트와 스케일링 이벤트가 실행 중인 워크로드를 방해하지 않도록 안정성을 개선했습니다.

- **무중단 업데이트**: 플랫폼 업데이트 및 롤링 재시작 중 서비스 영향을 최소화했습니다.
- **워커 정상 종료**: 스케일 인 및 업데이트 시 워커가 안전하게 종료되어 진행 중인 요청 유실을 방지합니다.
- **재시작 복원력**: 이벤트 처리 파이프라인이 컴포넌트 재시작 시에도 데이터 손실이나 순서 꼬임 없이 동작합니다.
- **HTTPS 도입**: 플랫폼 보안 기준으로 HTTPS 적용을 시작했습니다.

</Update>

## 과금 및 팀 협업

팀이 비용을 관리하고 리소스를 공유할 수 있도록 크레딧 기반 과금과 멀티 유저 협업 기능을 도입했습니다.

- **크레딧 기반 과금**: 선불 크레딧 시스템으로 사용량 비용을 관리하며 충전 및 잔액 추적이 가능합니다.
- **팀 초대**: 이메일 초대를 통해 팀원을 조직에 초대하고 공유 리소스에 함께 접근할 수 있습니다.
- **예약 용량**: 예약형(기간 기반)과 온디맨드 리소스를 구분하여 예측 가능한 비용 계획을 수립합니다.
- **과금 자동화**: 사용량 수집과 정산 처리가 스케줄 기반으로 자동 실행됩니다.

</Update>

## 플랫폼 기반 업그레이드

관측성, 캐싱, 배포 유연성을 위한 인프라 개선입니다.

- **사용자 워크로드 커스텀 메트릭**: 컨테이너별 메트릭(예: vLLM)을 수집/표시하며 메트릭 타겟을 설정할 수 있습니다.
- **헬스 체크 개선**: 부팅 시간이 긴 컨테이너의 준비 상태를 더 정확하게 감지하여 조기 종료를 방지합니다.
- **빠른 런타임 환경**: 더 가볍고 빠른 패키징 및 배포 구조로 시작 시간을 단축했습니다.
- **오프라인 배포 지원**: 제한된 네트워크 환경에서의 오프라인 운영 시나리오를 지원합니다.
- **모델 캐싱 개선**: HuggingFace 모델 캐시 공유 및 관리 방식을 개선하여 콜드 스타트를 단축했습니다.
- **안정적인 이벤트 캡처**: 오토스케일 및 운영 이벤트 캡처를 강화하여 더 안정적인 스케일링 결정을 지원합니다.

</Update>
