## October 2025

### Observability, dashboards, API keys, and UX upgrades

Major release bringing comprehensive observability, project dashboards, and security enhancements.

**Observability & Logs**
- **Log download and bulk export**: Download and analyze logs offline with single or batch export.
- **Advanced log filtering**: Regex-based and time-range filtering for faster troubleshooting.
- **Raw log (JSON) view**: Inspect unprocessed log data alongside the formatted view.
- **Scaling history logs**: Track autoscale events, timeout-triggered changes, and error-driven replica adjustments over time.

**Dashboards**
- **Project-level dashboards**: Monitor status and usage at the project level with dedicated dashboard views.
- **Enhanced filtering and sorting**: Find relevant metrics and endpoints faster with improved search and sort controls.
- **Extended request metrics**: Cumulative request counts, job error rates, and other operational metrics for quick health checks.

**UX Improvements**
- **Organization-level usage breakdown**: View usage and cost distribution by project, endpoint, and instance type.
- **Transaction history filtering**: Filter billing records by type and status.
- **Korean language support**: Full Korean UI now available.
- **First-time user onboarding**: Guided tutorial flow for new users.
- **In-product feedback**: Submit feedback directly from within the platform.

**Security & Access**
- **OpenAI-compatible API keys**: Issue API keys with OpenAI-compatible format and control per-key endpoint access.
- **HTTPS endpoints**: Enforced HTTPS for all inference endpoints.
- **Path-based endpoint addressing**: Moved from port-based to path-based routing with support for custom endpoint URLs.

## September 2025

### AirCloud General Availability — API-first platform with scheduled scaling

AirCloud is now generally available. This milestone release introduces API-first workflows, custom endpoints, and scheduled scaling.

- **Time-based scaling**: Schedule minimum replica counts or toggle autoscaling by time of day to match traffic patterns and reduce idle costs.
- **Custom endpoint URLs**: Use your own custom URL identifiers instead of system-generated IDs for cleaner integration.
- **API Playground**: Test and integrate APIs interactively through the built-in Playground experience.
- **One-click cluster deployment**: Reduce repetitive setup with templated, automated cluster deployments.
- **Actionable error messages**: Detailed error guidance on failure screens to cut mean-time-to-resolution.
- **Secure inference traffic (HTTPS)**: End-to-end HTTPS for inference requests to meet security and compliance requirements.
- **Resource management improvements**: Enhanced resource allocation and operational controls for production workloads.

## May 2025

### Usage insights, autoscaling usability, and scale tuning

New visibility into costs, smarter autoscaling controls, and performance hardening.

- **Usage & billing dashboards**: Monthly and daily usage visualization to track credit consumption and spending trends.
- **Autoscaling sensitivity controls**: Configure autoscale responsiveness (Heavy / Normal / Light) to match workload characteristics.
- **Deployment template reuse**: Copy existing deployment configurations to quickly create new deployments with the same settings.
- **Predictable scaling behavior**: Tuned scale-in/out policies and dampening for more consistent autoscaling under varying load patterns.
- **High-concurrency delivery**: Performance tuning for large-file transfers and concurrent request handling to improve stability under peak traffic.

## April 2025

### Zero-downtime hardening

Stability improvements for seamless updates and restarts.

- **Zero-downtime updates**: Reduced service impact during platform updates and rolling restarts.
- **Graceful worker drain**: Workers exit cleanly during scale-in and updates, preventing request drops.
- **Restart resilience**: Event processing pipelines now handle component restarts without data loss or ordering issues.
- **HTTPS adoption**: Began HTTPS rollout as the platform security baseline.

## February 2025

### Billing and team collaboration foundations

Introduced credit-based billing and multi-user collaboration.

- **Credits-based billing**: Manage usage costs through a prepaid credit system with top-up and balance tracking.
- **Team invites**: Invite team members to your organization via email for collaborative access to shared resources.
- **Reserved capacity**: Distinguish between reserved (term-based) and on-demand resources for predictable cost planning.
- **Billing automation**: Automated usage collection and settlement processing on scheduled intervals.

## January 2025

### Platform foundation upgrades

Infrastructure improvements for observability, caching, and deployment flexibility.

- **Custom metrics for user workloads**: Collect and display per-container metrics (e.g., vLLM) with configurable metric targets.
- **Improved health checks**: Better readiness detection for containers with long boot times to prevent premature termination.
- **Faster runtime environments**: Lighter, faster packaging and deployment structures for reduced startup times.
- **Offline-friendly deployment**: Support for restricted-network environments with offline operation scenarios.
- **Better model caching**: More efficient HuggingFace model cache sharing and management for faster cold starts.
- **Reliable event capture**: Hardened autoscale and operational event capture for more stable scaling decisions.
